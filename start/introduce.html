BUI 简介

开始BUI

本章主要解决一下问题：

  1. 如何去学习BUI？
  2. 我去哪里查找文档？
  3. 我为什么需要一个web服务器？
  4. 我如何参与BUI和贡献自己的力量？

文档

BUI 网站提供了丰富的Demo和API文档，这个文档中心页提供了4篇文章，供初学者入门使用。

导引

我们为开发者提供了大量的章节介绍BUI，一般来讲文档组织的顺序就是我们推荐的顺序。为了回答”我如何开始学习BUI？“，你应该从Hello BUI!开始，或者你已经使用过BUI或者看过本章，可以从后面的章节开始。

参考指南

更加详细的参考指南是API文档，更多的文档可能会在github上进行改进并放出

查看API

这里有BUI的全部API文档，这些文档是从代码注释中自动生成的，我们使用了js-duck,你也可以下载下源码，修改注释，生成自己的API文档

目前API的版本只有一个，还未有新的版本发布，所有的问题都会有积极的反馈，请将问题反馈到issue 列表中,如果未来我们提供新版本，会让用户方便的访问到各个版本的API。并标示清晰。

常见问题

这里有很多常见问题，可能对你有所帮助

API过期警告

在API中你可能会看到deprecation的警告，我们可能在未来更改对应的方法事件名称或者移除该方法。为了统一命名和使用的方便性。

使用一个web服务器

我们的代码或者demo需要在个服务器环境下才能正常运行，由于有些文档是.php文件，有些demo需要异步请求数据，所以建议用户下载到本地的demo在服务器下运行。

window 环境下可以使用IIS或者下载Easy PHP,具体搭建教程自己baidu

使用CDN

你或许想使用CDN上的BUI代码，服务器的稳定性和速度绝对有保障，我们提供了http和https2种协议的CDN路径：

  1. http 例如：http://g.tbcdn.cn/fi/bui/seed-min.js
  2. https 例如：https://s.tbcdn.cn/g/fi/bui/seed-min.js

具体的加载方式和需要引入的JS文件，请参看章节：BUI的安装以及加载

常见的错误汇总

使用过程中你可能会发现很多容易引起错误的地方，请提交的issue列表，或者查看其它人提交的的错误。

已知的文档问题

我们正在持续的改进我们的文档，也就是说我们知道目前有很多问题，已知的问题：

  • API 中的代码示例不够充分，很多特性未提供示例。
  • Demo 的数量依然不足，很多控件的特性和容易出问题的配置项，未提供demo
  • BUI 网站对低版本的ie支持不足

反馈

你可以将问题反馈到issue列表中,也可以去QQ群：138692365 ,旺旺群：778141976中跟其他使用者交流。

贡献代码和文档

我一个人的时间和精力毕竟有限，控件新功能和文档我希望大家能够积极参与进来，我们使用github来管理和提交代码，BUI 控件库和文档库都在上面

  1. API 文档库，你可以帮助我修改对应的模板，和完善代码的注释
  2. BUI 应用库，你可以在对应的应用中添加业务场景，和改善交互体验
  3. 指引文档，你可以提出你的建议，更希望能积极的参与进文档的编写过程中来
  4. 贡献patches和 bug fixes,请在github上按照提交规则进行
